¡Bienvenidos a la lección 2! Es hora de escribir código.
(Estar en CodePen.io, en una ventana nueva y vacía).

Muy bien, estamos en CodePen, en la ventana de HTML.
Toda página web, no importa si es Google o un proyecto pequeño, necesita una estructura básica.

(Empezar a escribir)
Primero, le decimos al navegador que esto es un documento HTML. Escribimos `<html>`.
(Escribir `<html>`)
Y como casi todas las etiquetas en HTML, necesitamos cerrarla. Escribimos `</html>`.
(Escribir `</html>`)
Cualquier cosa que sea parte de nuestra página web tiene que vivir DENTRO de estas dos etiquetas.

Dentro de `<html>`, la página se divide en dos partes MUY importantes: la "cabeza" y el "cuerpo".
(Escribir `<head>` y `</head>`)
El `<head>` (la cabeza) es para información *sobre* la página. Cosas que el usuario no ve directamente, como el título que aparece en la pestaña del navegador.
(Escribir dentro del head) `<title>Mi Primera Página</title>`
(Señalar la pestaña del navegador de CodePen) "Ven ese título allá arriba? Eso lo controla la etiqueta <title>."

Y luego, la parte más importante...
(Escribir `<body>` y `</body>` debajo del head)
El `<body>` (el cuerpo). ¡Aquí es donde ocurre toda la magia! Todo lo que ves en una página web (texto, imágenes, videos) vive DENTRO del `<body>`.

Vamos a añadir nuestro primer elemento. La etiqueta más común para texto es la etiqueta `<p>`, que significa "párrafo".
(Escribir dentro del body) `<p>¡Hola Mundo!</p>`
(Señalar la vista previa)
¡Y miren! Al instante, apareció "Hola Mundo" en nuestra vista previa. ¡Acaban de crear su primera línea de HTML!

Vamos a añadir otro párrafo.
(Escribir) `<p>Estoy aprendiendo a programar mi primera página web.</p>`
Perfecto.

### Errores Comunes (Extenso)
Ahora, te voy a enseñar el error #1 que TODOS cometen cuando empiezan.
(Ir al código y **borrar intencionalmente la etiqueta de cierre `</p>`** del primer párrafo).

`<p>Soy [Tu Nombre]`
`<p>Estoy aprendiendo a programar...</p>`
`<p>Mi comida favorita es la pizza.</p>`

(Señalar la vista previa)
"¿Ven qué pasó? Se ve... raro. Todo está en la misma línea, o el espacio es extraño. El navegador se acaba de confundir.
No supo dónde terminaba el primer párrafo, así que simplemente mezcló todo.
El navegador es muy literal. Si no le pones la etiqueta de cierre `</ap>`, él no sabe cuándo parar."

(Volver a poner el `</p>`).
"¡Y listo! Todo vuelve a la normalidad.
**Recuerda: ¡Casi toda etiqueta que abres (`<p>`), la debes cerrar (`</p>`)!**"
