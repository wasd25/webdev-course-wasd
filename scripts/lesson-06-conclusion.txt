Ahora, quiero hacer un repaso final de los **errores más comunes** que vimos. Si tu código no funciona, ¡99% seguro es una de estas 4 cosas!

(Mostrar una lista en pantalla)
1.  **En HTML: Olvidar una etiqueta de cierre.** (Mostrar el ejemplo de `<p>...<p>`). ¡Recuerden siempre cerrar sus etiquetas (`</p>`)! Si abres una puerta, ciérrala.
2.  **En HTML: Olvidar el `alt` en las imágenes.** Si la imagen se rompe, el usuario no sabrá qué había. ¡Siempre usen `alt`!
3.  **En CSS: Olvidar el punto y coma (`;`).** (Mostrar el ejemplo). Es el error más frustrante. Si lo olvidas, la siguiente línea de código CSS no funcionará. Es el punto al final de la oración.
4.  **En CSS: Olvidar el punto (`.`) para las clases.** Recuerda, `p` es para una etiqueta, `.perfil-card` es para una clase. El punto es la diferencia.

Si pueden recordar esos 4 errores, ¡ya están muy por delante de cuando empezamos!

¿Y ahora qué?
El viaje no termina aquí. Si te gustó esto, el siguiente paso es aprender más sobre CSS (conceptos como Flexbox y Grid, que usamos para centrar).
Y luego, el tercer gran lenguaje de la web: **JavaScript**.
Si HTML son los huesos y CSS es la ropa... JavaScript es el "cerebro". Es lo que añade la interactividad.

Un recurso increíble para seguir aprendiendo es **MDN Web Docs**. Búsquenlo en Google, es el mejor manual de referencia que existe.
