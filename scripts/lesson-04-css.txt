Muy bien, nuestra página tiene estructura, pero se ve... simple. ¡Vamos a arreglarlo con CSS! [cite: 46]

CSS funciona con "selectores". Un selector es cómo le decimos a CSS a qué etiqueta de HTML queremos aplicarle estilo.
¿Quieren cambiar TODA la página? Seleccionamos la etiqueta `<body>`.
(Escribir en la ventana de CSS)
`body {`
`  background-color: #f0f0f0; /* Un gris muy claro */`
`}`
¡Boom! El fondo cambió. Ahora, quiero que todos mis títulos `<h1>` sean azules.
(Escribir)
`h1 {`
`  color: blue;`
`  text-align: center; /* ¡Y vamos a centrarlo! */`
`}`
La sintaxis es: `selector { propiedad: valor; }`[cite: 111].

**ERRORES COMUNES (5-6 minutos)**
Aquí es donde la gente más se equivoca en CSS.
Error #1: Olvidar el punto y coma (`;`).
(Ir al código CSS y **borrar intencionalmente el punto y coma** después de `18px`).
`p {`
`  font-size: 18px`
`  font-family: Arial, sans-serif;`
`}`

"¡Oh no! ¿Vieron eso? El tamaño de la letra (`font-size`) funcionó, ¡pero la fuente (`font-family`) no! El navegador se confundió. ¿Por qué? Porque en CSS, cada regla *debe* terminar con un punto y coma (`;`). Si no lo pones, la siguiente regla no funciona."
(Agregar el punto y coma de nuevo). "¡Ahí está!"

Error #2: Olvidar una llave de cierre (`}`).
(Borrar la llave de cierre `}` del selector `p`)
"¡Miren! ¡Ahora todo mi CSS se rompió! ¡Incluso el estilo del `h1` desapareció! ¿Por qué? Porque al borrar la llave de cierre, el navegador piensa que todo lo que sigue es parte del selector `p`."
(Poner la llave `}` de nuevo). "¡Perfecto!"
